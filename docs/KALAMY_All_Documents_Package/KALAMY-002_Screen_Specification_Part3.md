# KALAMY-002
Screen Specification

## الجزء الثالث: تنفيذ التمارين والجلسات والتقارير

### SCR-008 - تنفيذ التمرين

الغرض: تشغيل التمرين وتسجيل أداء المستفيد.

| الحقل | النوع | إلزامي | ملاحظات |
| --- | --- | --- | --- |
| اسم التمرين | Label | نعم | يعرض تلقائياً |
| تعليمات | Text | نعم | قبل البدء |
| مؤقت | Timer | لا | حسب التمرين |
| تسجيل الأداء | Audio/Text | حسب التمرين | اختياري أو إلزامي |

#### الأزرار

- بدء

- إيقاف مؤقت

- إنهاء

- إرسال النتيجة

#### قواعد العمل

- لا يمكن إنهاء التمرين دون حفظ النتيجة إذا كان التمرين إلزامياً.

- يتم تحديث نسبة الإنجاز تلقائياً.

API: POST /api/v1/progress

Navigation: العودة إلى لوحة التقدم

### SCR-009 - الجلسة العلاجية

الغرض: توثيق الجلسة العلاجية الحضورية أو عن بعد.

| الحقل | النوع | إلزامي | ملاحظات |
| --- | --- | --- | --- |
| نوع الجلسة | Dropdown | نعم | حضوري/عن بعد |
| الملاحظات | TextArea | نعم |  |
| الواجب المنزلي | TextArea | لا |  |
| موعد المراجعة | Date | لا |  |

#### الأزرار

- حفظ

- إنهاء الجلسة

#### قواعد العمل

- يجب حفظ ملاحظات الجلسة قبل الإغلاق.

API: PUT /api/v1/sessions/{id}

Navigation: العودة إلى لوحة الأخصائي

### SCR-010 - لوحة متابعة التقدم

الغرض: عرض تقدم المستفيد والمؤشرات العلاجية.

| الحقل | النوع | إلزامي | ملاحظات |
| --- | --- | --- | --- |
| نسبة الإنجاز | Chart | نعم |  |
| آخر التمارين | List | نعم |  |
| مؤشرات الأداء | Cards | نعم |  |

#### الأزرار

- تحديث

- عرض التقرير

#### قواعد العمل

- تُحدَّث البيانات مباشرة بعد أي نشاط علاجي.

API: GET /api/v1/progress/{patientId}

Navigation: الانتقال إلى شاشة التقرير

### SCR-011 - التقارير

الغرض: عرض وإصدار التقارير العلاجية.

| الحقل | النوع | إلزامي | ملاحظات |
| --- | --- | --- | --- |
| نوع التقرير | Dropdown | نعم |  |
| الفترة | Date Range | نعم |  |
| صيغة الإخراج | Dropdown | لا | PDF افتراضياً |

#### الأزرار

- عرض

- تنزيل PDF

- مشاركة

#### قواعد العمل

- التقرير المعتمد لا يُعدل ويُنشأ إصدار جديد عند التحديث.

API: GET /api/v1/reports/{patientId}

Navigation: العودة إلى لوحة التقدم
