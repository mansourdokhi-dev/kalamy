## 1. هدف الوثيقة

تحدد هذه الوثيقة نطاق النسخة الأولى القابلة للتنفيذ من برنامج كلامي، بحيث يستطيع فريق التصميم والبرمجة البدء في بناء MVP واضح ومحدود، دون الدخول في كامل المنصة المؤسسية من البداية. الهدف هو إطلاق نسخة عملية تثبت القيمة العلاجية والتجارية، ثم يتم التوسع لاحقًا.

## 2. تعريف MVP

MVP في برنامج كلامي هو أول إصدار رقمي قابل للاستخدام من قبل المستفيد، ولي الأمر، والأخصائي، ويغطي رحلة علاج التلعثم الأساسية: التسجيل، التقييم الأولي، إنشاء الخطة العلاجية، تقديم جلسات/تمارين رقمية، المتابعة، التقارير، والإشراف السريري.

## 3. نطاق النسخة الأولى

| البند | التفصيل |
| --- | --- |
| داخل النطاق | تسجيل المستخدمين، ملف المستفيد، تقييم أولي، خطة علاج، مكتبة تمارين، جلسات رقمية، متابعة تقدم، تقارير، لوحة أخصائي، إدارة بسيطة. |
| خارج النطاق | تكامل المستشفيات، الذكاء الاصطناعي المتقدم، التأمين، أجهزة قياس خارجية، Marketplace، تعدد المنظمات المعقد. |
| الجمهور المستهدف | الأطفال والبالغون ممن لديهم تلعثم، أولياء الأمور، أخصائيو التخاطب، مشرف سريري، مدير نظام. |
| الهدف التشغيلي | تجربة علاجية رقمية قابلة للقياس، يمكن تشغيلها تجريبيًا على مجموعة محدودة ثم تطويرها تدريجيًا. |

## 4. الأدوار الأساسية

| Role | المسمى | الصلاحيات الرئيسية |
| --- | --- | --- |
| Patient | المستفيد | إنشاء حساب، أداء التمارين، متابعة التقدم، حضور الجلسات عن بُعد. |
| Caregiver | ولي الأمر | متابعة الطفل، الاطلاع على الواجبات، استلام التعليمات والتقارير. |
| Clinician | الأخصائي | التقييم، الخطة العلاجية، إدارة الجلسات، مراجعة التقدم. |
| Supervisor | المشرف السريري | مراجعة جودة الخطط والجلسات، اعتماد البروتوكولات. |
| Admin | مدير النظام | إدارة المستخدمين، المحتوى، الصلاحيات، التقارير العامة. |

## 5. الرحلة الأساسية للمستفيد

| # | المرحلة | الوصف |
| --- | --- | --- |
| 1 | إنشاء حساب | المستفيد أو ولي الأمر يسجل البيانات الأساسية. |
| 2 | إكمال الملف | إدخال العمر، الجنس، تاريخ الحالة، شدة التلعثم، بيانات التواصل. |
| 3 | التقييم الأولي | استبيان + مقطع عينة كلام + تقييم أخصائي عند الحاجة. |
| 4 | الخطة العلاجية | تحديد المرحلة العلاجية، الأهداف، التمارين، مدة البرنامج. |
| 5 | تنفيذ التمارين | المستفيد يؤدي التمارين حسب الجدول ويُسجل النتائج. |
| 6 | المتابعة | النظام يعرض نسبة الإنجاز والتقدم والتنبيهات. |
| 7 | تقرير مرحلي | الأخصائي وولي الأمر يطلعان على تقرير التقدم. |
| 8 | تعديل الخطة | الأخصائي يعدل البرنامج بناءً على الأداء. |

## 11. مؤشرات النجاح في MVP

| --- | --- | --- |
| Completion Rate | نسبة إكمال التمارين | ≥70% |

| User Satisfaction | رضا المستفيد/ولي الأمر | ≥80% |

## 13. معايير قبول MVP

- يمكن للمستفيد إنشاء حساب والدخول للمنصة.

- يمكن إنشاء ملف مستفيد مكتمل.

- يمكن تنفيذ تقييم أولي وتخزين نتيجته.

- يمكن للأخصائي إنشاء خطة علاجية وربط تمارين بها.

- يمكن للمستفيد تنفيذ تمارين وتسجيل التقدم.

- يمكن للأخصائي متابعة الأداء وإضافة ملاحظات.

- يمكن إصدار تقرير تقدم بسيط.

- الصلاحيات الأساسية تعمل حسب الدور.

- النظام مستقر في بيئة تجريبية محدودة.

- لا توجد ثغرات أمنية حرجة قبل الإطلاق التجريبي.
