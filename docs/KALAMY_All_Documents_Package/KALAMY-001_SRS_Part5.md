# KALAMY-001
Software Requirements Specification (SRS)

## الجزء الخامس: المتطلبات غير الوظيفية ومعايير الجودة

### 1. الأداء

- زمن استجابة معظم العمليات أقل من ثانيتين.

- دعم المستخدمين المتزامنين وفق خطة السعة.

- حفظ العمليات دون فقدان البيانات.

### 2. الأمان

- تشفير البيانات أثناء النقل والتخزين.

- مصادقة آمنة.

- صلاحيات حسب الدور.

- سجل تدقيق لجميع العمليات الحساسة.

### 3. التوفر والاستمرارية

- استهداف توفر 99.9%.

- نسخ احتياطي دوري.

- خطة استعادة عند الطوارئ.

### 4. سهولة الاستخدام

- واجهة عربية أولاً مع دعم الإنجليزية.

- تصميم متوافق مع الهواتف والأجهزة اللوحية والويب.

- الالتزام بمبادئ إمكانية الوصول.

### 5. الخصوصية

- حماية البيانات الصحية.

- تسجيل موافقات المستخدم عند الحاجة.

- تقييد الوصول للبيانات حسب الصلاحيات.

### 6. الجودة

- اختبارات وحدة وتكامل وقبول.

- مراجعة الكود.

- مراجعة سريرية للوظائف العلاجية.

### 7. معايير القبول

- نجاح جميع الاختبارات الحرجة.

- عدم وجود أخطاء حرجة قبل الإطلاق.

- اعتماد المالك والأخصائي والمبرمج.

### 8. خطة التسليم

- تسليم التصميم.

- تسليم البرمجة.

- الاختبارات.

- الإطلاق التجريبي.

- الإطلاق الرسمي.

### الملحق

تمثل الأجزاء الخمسة من وثيقة KALAMY-001 المرجع التنفيذي الأساسي للإصدار الأول من المنصة، وتعد أساساً لبدء التصميم والبرمجة مع إمكانية التوسع لاحقاً في الوثائق التفصيلية.
