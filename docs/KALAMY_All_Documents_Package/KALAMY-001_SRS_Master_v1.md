# KALAMY-001
Software Requirements Specification (SRS)
Master Document

## حالة الوثيقة

هذه هي النسخة الأساسية الجاهزة لبدء التصميم والتحليل الفني للإصدار الأول (MVP) من برنامج كلامي. تشكل المرجع الرسمي الذي سيبنى عليه التصميم وقاعدة البيانات وواجهات API.

## 1. أهداف المنتج

- منصة علاج رقمي للتلعثم.

- إدارة رحلة المستفيد كاملة.

- دعم العلاج الحضوري وعن بعد.

- قياس النتائج السريرية.

## 2. المستخدمون

- المستفيد

- ولي الأمر

- الأخصائي

- المشرف السريري

- مدير النظام

## 3. الوحدات الرئيسية

- الحسابات

- الملف السريري

- التقييم

- الخطة العلاجية

- التمارين

- الجلسات

- التقارير

- الإدارة

## 6. مخرجات الوثيقة

- تعتمد هذه الوثيقة كأساس للتصميم.

- أي تفصيل إضافي ينتقل إلى الوثائق التالية: Screen Specification, Database Dictionary, API Specification.

## اعتماد الوثيقة

هذه الوثيقة مناسبة لبدء تحليل التصميم وتقسيم المشروع إلى وحدات عمل. أما التنفيذ البرمجي الكامل فيعتمد كذلك على الوثائق التالية: KALAMY-002 (Screen Specification)، KALAMY-003 (Database Dictionary)، KALAMY-004 (API Specification).
