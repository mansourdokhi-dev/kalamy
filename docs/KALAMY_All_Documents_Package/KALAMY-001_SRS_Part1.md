# KALAMY-001
Software Requirements Specification (SRS)

## الجزء الأول: مقدمة المنتج والمتطلبات الوظيفية الأساسية

### 1. هدف الوثيقة

تمثل هذه الوثيقة المرجع التنفيذي الرئيسي لفريق التصميم والبرمجة لبناء الإصدار الأول من منصة كلامي، وتصف الوظائف المطلوبة بصورة قابلة للتنفيذ.

### 2. نطاق الإصدار الأول (MVP)

- إدارة المستخدمين والحسابات.

- ملف المستفيد.

- التقييم الأولي.

- الخطة العلاجية.

- مكتبة التمارين.

- جلسات العلاج.

- متابعة التقدم.

- التقارير.

- لوحة الأخصائي.

- لوحة الإدارة.

### 3. الأدوار

| الدور | الوصف | أهم الصلاحيات |
| --- | --- | --- |
| المستفيد | المريض | تنفيذ التمارين ومتابعة التقدم |
| ولي الأمر | متابعة الطفل | إدارة الحساب والواجبات |
| الأخصائي | أخصائي التخاطب | التقييم والخطة والجلسات |
| المشرف | مشرف سريري | مراجعة الجودة |
| مدير النظام | الإدارة | إدارة النظام |

### FR-001 - إنشاء حساب

الوصف: إنشاء حساب جديد للمستفيد أو ولي الأمر.

المدخلات:

- الاسم

- رقم الجوال

- البريد الإلكتروني

- كلمة المرور

قواعد التحقق:

- رقم الجوال فريد

- كلمة المرور 8 أحرف على الأقل

- البريد الإلكتروني صحيح

المخرجات:

- إنشاء الحساب

- إرسال رمز تحقق

- إنشاء الملف الأساسي

Business Rule: إذا كان العمر أقل من 18 سنة فيلزم ربط الحساب بولي أمر.

API: POST /api/v1/auth/register

Database: Users, Patients, Guardians

### FR-002 - تسجيل الدخول

الوصف: السماح للمستخدم بالدخول إلى النظام.

المدخلات:

- الجوال أو البريد

- كلمة المرور

قواعد التحقق:

- الحساب فعال

- بيانات الدخول صحيحة

المخرجات:

- إصدار رمز دخول

- تحميل الصلاحيات

Business Rule: بعد 5 محاولات فاشلة يتم إيقاف المحاولة مؤقتاً.

API: POST /api/v1/auth/login

Database: Users, Roles

### FR-003 - إنشاء ملف مستفيد

الوصف: إنشاء ملف سريري للمستفيد.

المدخلات:

- البيانات الشخصية

- التاريخ المرضي

- بيانات التواصل

قواعد التحقق:

- جميع الحقول الإلزامية مكتملة

المخرجات:

- حفظ الملف

- إتاحة التقييم الأولي

Business Rule: لا يمكن بدء التقييم دون اكتمال الملف.

API: POST /api/v1/patients

Database: Patients
