# §101 + §103 Notification Triggers Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Notify a patient when their sample-recording stage opens (§101), and notify every specialist when a sample newly arrives in the review queue (§103).

**Architecture:** Two new `NotificationType` values plus their templates (Task 1), a shared `getNotificationContext` utility extracted from `specialist-review.service.ts` to avoid a third/fourth copy of the same lookup (also Task 1), then one new call site each in `TrainingCyclesService.recordTrainingEvent` (Task 2) and `SamplesService.submitSample` (Task 3).

**Tech Stack:** NestJS, Prisma, Jest + Supertest (e2e against a real Postgres, no mocks).

## Global Constraints

- This is the first task in the notifications work that needs a Prisma migration — the two new `NotificationType` enum values.
- §103 notifies `CLINICIAN` only, not `ADMIN` — matches the existing precedent for the 24h-escalation notification (`ADMIN` already sees everything via admin reporting).
- Every notification call site in this codebase wraps the `notificationsService` call in `try/catch` with `this.logger.error(...)` on failure — a notification failure must never break or mask the business operation it's attached to (established convention, see `specialist-review.service.ts`'s three existing call sites). Both new call sites in this plan follow the same pattern.
- Message text stays hardcoded Arabic in `NOTIFICATION_TEMPLATES` — no admin-configurable content in this pass (that's §99, separately deferred).
- Run unit tests with `npm test` and e2e tests with `npm run test:e2e` from `backend/`.

```typescript
import { PrismaService } from '../../prisma/prisma.service';

export async function getNotificationContext(
  prisma: PrismaService,
  cycle: { patientProfileId: string; levelId: string },
): Promise<{ patientName: string; levelName: string }> {
  const [patientProfile, level] = await Promise.all([
    prisma.patientProfile.findUniqueOrThrow({ where: { id: cycle.patientProfileId } }),
    prisma.level.findUniqueOrThrow({ where: { id: cycle.levelId } }),
  ]);
  return { patientName: patientProfile.fullName, levelName: level.name };
}
```
