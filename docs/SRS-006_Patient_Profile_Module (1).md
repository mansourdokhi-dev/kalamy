# SRS-006
Patient Profile Module

## 1. الهدف

تحديد متطلبات إنشاء وإدارة الملف السريري للمستفيد بحيث يكون المرجع الأساسي لجميع عمليات التقييم والعلاج.

## 2. البيانات الأساسية

- الرقم التعريفي للمستفيد

- الاسم الرباعي

- الجنس

- تاريخ الميلاد

- الهوية/الإقامة

- رقم الجوال

- البريد الإلكتروني

- العنوان

- بيانات ولي الأمر

- جهة الإحالة

## 3. البيانات السريرية

- سبب الإحالة

- التشخيص الأولي

- التاريخ المرضي

- الأدوية

- الحساسية

- الأمراض المصاحبة

- التاريخ الأسري

- الموافقات الإلكترونية

## 4. User Story

كمتخصص تخاطب، أريد إنشاء ملف سريري مكتمل للمستفيد حتى أتمكن من إجراء التقييم ووضع الخطة العلاجية.

## 5. قواعد العمل

- لا يمكن بدء التقييم قبل اكتمال الحقول الإلزامية.

- يجب ربط ولي أمر إذا كان العمر أقل من 18 سنة.

- يحفظ سجل التعديلات كاملاً.

- لا يسمح بحذف الملف السريري؛ يتم تعطيله فقط عند الحاجة.

## 6. واجهات API

| الطريقة | المسار | الوصف |
| --- | --- | --- |
| POST | /api/v1/patients | إنشاء ملف |
| GET | /api/v1/patients/{id} | عرض الملف |
| PUT | /api/v1/patients/{id} | تحديث الملف |
| GET | /api/v1/patients | البحث في الملفات |

## 7. معايير القبول

- إنشاء الملف عند اكتمال البيانات.

- منع التكرار حسب الهوية أو السجل المحدد.

- ظهور الملف في لوحة الأخصائي.

- إتاحة الانتقال إلى شاشة التقييم بعد اكتمال الملف.

## 8. حالات الاختبار

- إنشاء ملف كامل.

- إنشاء ملف مع نقص بيانات إلزامية.

- تحديث بيانات.

- البحث عن مستفيد.

- ربط ولي أمر.
