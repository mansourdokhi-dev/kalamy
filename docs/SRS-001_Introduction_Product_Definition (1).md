# KALAMY-001
Software Requirements Specification (SRS)
SRS-001 – Introduction & Product Definition

## 1. الغرض من الوثيقة

تُعد هذه الوثيقة المرجع الرسمي لمتطلبات برنامج كلامي. وهي موجهة لفريق إدارة المشروع، ومصممي UX/UI، ومطوري Frontend وBackend، ومهندسي الجودة والاختبارات.

## 2. أهداف المنتج

- توفير منصة رقمية متكاملة لعلاج التلعثم.

- إدارة رحلة المستفيد من التسجيل حتى إنهاء البرنامج.

- دعم العلاج الحضوري والعلاج عن بعد.

- تمكين الأخصائي من متابعة النتائج واتخاذ القرار السريري.

- إصدار تقارير قابلة للقياس.

## 3. نطاق الإصدار الأول (MVP)

- إدارة الحسابات.

- الملف السريري.

- التقييم الأولي.

- الخطة العلاجية.

- التمارين الرقمية.

- الجلسات العلاجية.

- التقارير.

- الإشعارات.

- لوحة الأخصائي.

- لوحة الإدارة.

## 4. أصحاب المصلحة

| الفئة | الدور | الهدف |
| --- | --- | --- |
| المستفيد | استخدام البرنامج | تحسين الطلاقة |
| ولي الأمر | المتابعة | دعم الالتزام |
| الأخصائي | العلاج | إدارة الخطة |
| المشرف | الجودة | الإشراف السريري |
| مدير النظام | الإدارة | تشغيل المنصة |

## 6. الافتراضات

- المنصة تدعم الويب وiOS وAndroid.

- اللغة العربية هي اللغة الأساسية مع إمكانية إضافة الإنجليزية.

- جميع البيانات الصحية تخضع لصلاحيات الوصول.

## 8. خطة استكمال SRS

سيتم استكمال وثيقة SRS على ملفات متتابعة تغطي المتطلبات الوظيفية بالتفصيل، ثم دمجها في ملف واحد يمثل المرجع النهائي للتطوير.
