# SRS-007
Assessment Module

## 1. الهدف

تحديد متطلبات وحدة التقييم الأولي والدوري للمستفيد بحيث يتم إنشاء خط أساس سريري يمكن الاعتماد عليه في بناء الخطة العلاجية وقياس التحسن.

## 2. أنواع التقييم

- التقييم الأولي

- إعادة التقييم الدوري

- التقييم الختامي

- استبيانات المستفيد وولي الأمر

- تسجيلات صوتية قبل وبعد العلاج

## 3. البيانات المطلوبة

- التاريخ المرضي

- درجة شدة التلعثم

- المواقف الأكثر صعوبة

- مستوى القلق المرتبط بالكلام

- الأهداف العلاجية الأولية

- ملاحظات الأخصائي

## 4. User Story

كأخصائي تخاطب، أريد إجراء تقييم معياري للمستفيد حتى أستطيع تحديد المرحلة العلاجية المناسبة.

## 5. قواعد العمل

- لا يبدأ التقييم قبل اكتمال الملف السريري.

- يجب حفظ جميع نتائج التقييم مع التاريخ والإصدار.

- لا تُنشأ الخطة العلاجية إلا بعد اعتماد التقييم.

- يمكن مقارنة نتائج إعادة التقييم مع خط الأساس.

## 6. واجهات API

| الطريقة | المسار | الوصف |
| --- | --- | --- |
| POST | /api/v1/assessments | إنشاء تقييم |
| GET | /api/v1/assessments/{id} | عرض تقييم |
| PUT | /api/v1/assessments/{id} | تحديث تقييم |
| GET | /api/v1/patients/{id}/assessments | قائمة التقييمات |

## 7. معايير القبول

- حفظ التقييم بنجاح.

- إظهار نتيجة التقييم في ملف المستفيد.

- إتاحة إنشاء الخطة العلاجية بعد الاعتماد.

- حفظ جميع النسخ السابقة للتقييم.

## 8. حالات الاختبار

- إجراء تقييم أولي.

- حفظ تقييم ناقص (يجب الرفض).

- اعتماد تقييم.

- إعادة تقييم ومقارنة النتائج.
