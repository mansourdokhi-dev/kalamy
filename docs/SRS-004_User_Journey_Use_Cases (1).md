# SRS-004
User Journey & Use Case Specification

## 1. رحلة المستفيد

| المرحلة | اسم المرحلة | الإجراء | المخرج |
| --- | --- | --- | --- |
| 1 | إنشاء حساب | إدخال البيانات والتحقق من OTP | حساب مفعل |
| 2 | إكمال الملف | إدخال البيانات الشخصية والسريرية | ملف مكتمل |
| 3 | التقييم الأولي | الاستبيانات وعينة الصوت | تقييم معتمد |
| 4 | الخطة العلاجية | اعتماد الخطة والتمارين | خطة فعالة |
| 5 | تنفيذ التمارين | أداء التمارين اليومية | تحديث التقدم |
| 6 | الجلسات | حضور الجلسات وتوثيقها | نتائج الجلسة |
| 7 | المتابعة | عرض الرسوم البيانية والتقارير | استمرار أو إنهاء |

## 2. Use Cases الأساسية

| الرمز | الحالة | المستخدم |
| --- | --- | --- |
| UC-001 | إنشاء حساب | المستفيد |
| UC-002 | تسجيل الدخول | جميع المستخدمين |
| UC-003 | إجراء تقييم | الأخصائي |
| UC-004 | إنشاء خطة علاجية | الأخصائي |
| UC-005 | تنفيذ تمرين | المستفيد |
| UC-006 | توثيق جلسة | الأخصائي |
| UC-007 | عرض تقرير | المستفيد/الأخصائي |

## 3. السيناريو الرئيسي

- يسجل المستفيد حساباً.

- يكمل الملف الشخصي.

- يعتمد الأخصائي التقييم.

- تنشأ الخطة العلاجية.

- يبدأ تنفيذ التمارين.

- تنعقد الجلسات العلاجية.

- تصدر التقارير ويقاس التقدم.

## 4. السيناريوهات البديلة

- فشل التحقق من OTP → إعادة إرسال الرمز.

- عدم اكتمال الملف → منع بدء التقييم.

- عدم اعتماد التقييم → منع إنشاء الخطة.

- غياب المستفيد → إعادة جدولة الجلسة.

## 5. قواعد العمل المرتبطة

- لا ينتقل المستفيد للمرحلة التالية قبل اعتماد السابقة.

- تحدث مؤشرات الأداء بعد كل نشاط.

- جميع الأحداث تسجل في Audit Log.
